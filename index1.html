Technical Report for NePr - A Platform to Track and Analyze the Provenance of Online News Articles

Authors: Opariuc Rares-Ioan, Tablan Andrei Razvan

Introduction

This report provides a comprehensive technical overview of NePr, a web platform designed to track, model, and manage the provenance of online newspaper articles, including multimedia content and language variants.

System Architecture

The architecture of NePr is designed using a microservices approach. The application integrates RDF data models, SPARQL queries, and uses cloud storage for RDF data. The system also leverages APIs from Wikidata and DBpedia.

Internal Data Structures and Models

NePr manages article metadata using structured data models based on standards like DCMI, ITPC, and the Social Semantic Web Thesaurus. RDF and schema.org vocabularies are utilized to describe resources semantically.

API Implementation

The REST and SPARQL-based API enables querying articles by provenance data, language, and media type. An OpenAPI specification outlines endpoints that allow access to articles, metadata, and recommendations.

RDF-based Knowledge Model

The RDF knowledge model uses schema.org and additional ontologies to structure provenance and content metadata. RDFa is embedded in article pages, while SPARQL queries enable complex searches.

Use of External Data and SPARQL Queries

NePr integrates data from Wikidata and DBpedia to enrich content provenance. Sample SPARQL queries retrieve articles based on language, length, or topic, and results conform to linked data principles.

User Guide

This section provides an overview of using the platform, including three case studies:

  1. Querying articles by specific provenance attributes.
  2. Finding multimedia content tagged with particular metadata.
  3. Viewing recommendations based on recent news and topics.
